# Визуализация китайского языка в виде графа

## Abstract

Китайский язык имеет в свой структуре много логики в силу особенностей своего развития. Я изучал китайский язык около двух лет, и сейчас могу сказать, что его можно довольно эффективно исследовать, делая акцент на отдельных иероглифах. Это наводит на мысль, что было бы удобно визуализировать всю систему иероглифов для более хорошего их изучения. Само по себе _визуализировать язык_ — звучит очень интересно и является сложной задачей для языков нашей языковой группы, но не в случае китайского языка. Потому я решил создать граф, устроенный просто: вершинами являются иероглифы, а рёбрами отмечаются любые связи между ними, которые захотим.

Естественно, хотелось бы, чтобы вершина содержала не просто один символ, но и ещё какую-то информацию для изучения. (Например: список синонимов, уровень изучения (HSK), рекомендации по запоминанию (мнемоника), ссылки на этот иероглиф в словарях, сложность написания (количество черт) и т.д.)

Более того, идею можно распространить на любой другой язык и, возможно, получить какой-то интересный результат.

## Idea

Визуализируем китайский язык с помощью графа.

Устройство графа таково:

1. Вершинами являются иероглифы; в вершинах также хранится дополнительная информация о них.
2. Граф полный: между каждой парой иероглифов существует связь, она характеризуется различными характеристики: синонимичность, встречаемость в одном предложении.

Граф будет визуализироваться с помощью matplotlib. Далее иероглифы с высокими характеристиками связи между друг другом будем называть _близкими_, так как логично отображать эти иероглифы рядом. Основные аспекты расположения графа и визуализации:

1. Граф располагается в пространстве V_{3}, чтобы можно было избежать пересечений рёбер.
2. Отображаются не все рёбра, а только те, которые проходят некий фильтр, что позволит отображать определённые связи.
3. Отображаются не все вершины. _Можно создать режимы просмотра графа: отображать только близкие иероглифы к определённому, отображать иероглифы до уровня HSK$n или все иероглифы._
3. Вершины располагаются как? Можно попытаться группировать близкие иероглифы в кластеры, но пока undefined.

(+) Также было бы интересно сделать граф интерактивным: пусть выделен некоторый _центральный_ иероглиф и отображаются только близкие к нему иероглифы. На любой отображаемый иероглиф можно нажать, после чего центральный иероглиф поменяется на выбранный.

(+) Также можно отображать вершины и рёбра графа необычным образом в зависимости от его характеристик. Например, цвета иероглифов (вершин) по их части речи (в китайском часть речи обычно undefined, но это пример).

## Plan

**O.** Написание и улучшение README. (iter 0)

**I.** Сбор информации об иероглифах. (iter 1)

Нужно создать базу иероглифов с необходимой информацией. Каждый иероглиф будет иметь свой файл .csv или .json.

Я реализую и сравню целых три (два с половиной) подхода:

1. Сбор информации в словарях. В китайских словарях есть безгранично много информации о иероглифах, но хочется использовать сразу русские, чтобы не ошибаться с переводом (и в принципе не тратить время на переводы). В них также можно найти и уровни HSK, но, кажется, только словарей будет недостаточно для определения уровня.
2. Генерация файлов через API ИИ! Как минимум из интереса, для сравнения с первым подходом. Полученная информация будет также нужна. (Хотелось бы использовать DeepSeek, сделанный в Китае, но такое  использование его API будет для нас дорогим.) Для этого будет создан отдельный модуль.
3. То же самое, но через графический интерфейс с DeepSeek! (Бесплатный способ достать из него информацию — имитировать поведение человека и т.д., копируя его ответы. Это довольно необычно и сложно, но интересно для сравнения с первыми двумя подходами.) Для этого также будет создан отдельный модуль.

**II.** Построение графа и визуализация. (iter 2)

Для этого буду использовать модуль matplotlib. Он позволит визуализировать граф в пространстве.

Также, вероятно, будут добавлены кнопки и ползунки для доступа к фильтру рёбер и другому.

Дополнительно ко всему этому нужно сначала придумать, как именно располагать вершины графа, что является нетривиальной задачей (идеального расположения вершин по любому шаблону ("чем ближе значения иероглифов, тем ближе вершины") почти никогда не существует).
